# スライド構成・コンテンツルール

## 基本構成（5〜9枚）

| Slide | 役割 | 内容 |
|-------|------|------|
| 1 | フック | テーマの提示 + 目次（何が含まれるか） + 「スワイプ →」誘導 |
| 2〜N | Q&A / メニュー / あるある | 悩み→回答 or 具体的な情報を1スライド1ポイントで |
| N+1 | 応用・選択余地 | 慣れてきたらカスタマイズできることを伝える（任意） |
| 最後 | まとめ | ポイント要約 + 保存誘導 + コメントCTA |

- 5〜7枚が基本 7〜9枚も有効（2025年トレンド: 情報密度の高いスライドは保存率UP）
- 10枚以上は離脱率が急増するため非推奨
- テーマの情報量に応じて枚数を柔軟に調整する

## Slide 1 フック戦略（3秒ルール）

Slide 1はサムネイル兼フック。3秒以内にスワイプさせる:
- **3〜5語の強いヘッドライン**: 文章ではなくインパクトのある短文
- **問いかけ or 変化の提示**: 「〜な人へ」「〜は全員通る道」
- **Slide 1にはフック文だけ**: 結論・要約・目次・内容のキーワードは一切入れない
- **目次・要約は絶対NG**: 「来ただけで勝ち / 1種目でOK」のような内容の要約を入れると中身を見る動機がなくなる
- **許容されるのはフック文 + 「スワイプ →」のみ**: それ以外の補助テキストは不要
- **「スワイプ →」を下部に**: 次のスライドがあることを明示

## コンテンツルール

- **Q&A形式**: 1問1答で具体的な悩みに具体的に答える Q.を使う場合は回答の先頭に必ずA.をつける（例: 「A. 食事が8割」）
- **浅い回答NG**: 「なぜそうなるか」「具体的にどうするか」まで踏み込む
- **強い言葉NG**: 「〜しろ」「〜やれ」→ 「〜してみて」「〜するといいよ」
- **煽り口調NG**: 「毎日それ食べられる?」「それでいいの?」のような挑発的な問いかけは絶対使わない
- **上から目線NG**: 「〜すべき」「〜しないとダメ」→ 「〜くらいの感覚でOKだよ」「〜がおすすめ」
- **トーン: 初心者に優しく教える友達**: 先輩風でもインフルエンサー風でもなく 同じ目線で寄り添う
- **断定NG**: 「これが正解」「迷わなくて良い」→ 「参考になったら嬉しい」「少しでも助けになれば」
- **句読点なし**: 全スライド共通で句読点は使わない
- **宣伝臭NG**: アプリ等の宣伝目的にしない 価値提供が最優先
- **1スライド12〜40文字**: 多すぎると読まれない（ただしメニュー系は種目リストのため例外）
- **ネガティブ表現NG**: ポジティブな文脈でも「気持ち悪い」等のネガティブな言葉は使わない
- **選択余地を与える**: 「これだけでいい」等の断定ではなく「参考にしてみてね」「自分のペースで」
- **深い内容にする**: 表面的な一言ではなく 3行程度の説明で「なぜ」「どう変わるか」まで踏み込む
- **強調行を入れる**: 各スライドで最も伝えたい1行をティール色で強調（全部白はNG）

## コンテンツの型

テーマに応じて最適な型を選ぶ:

| 型 | 構成例 | 適するテーマ |
|---|--------|------------|
| Q&A型 | フック + Q&A×3〜4 + まとめ | 悩み解決・豆知識 |
| メニュー型 | フック + メニュー×3〜6 + 応用 + まとめ | トレーニングメニュー・食事 |
| タイムライン型 | フック + 週ごと×3〜4 + まとめ | 変化の記録・経過報告 |
| あるある型 | フック + あるある×3〜4 + まとめ | 共感・コメント誘導 |

## アルゴリズム意識

- **Slide 1→2の遷移率が最重要**: フックで確実にスワイプさせる
- **スライド内テキストはTikTok検索にインデックスされる**: 検索キーワードを自然に含める
- **保存率 > いいね率**: 保存は2026年時点でいいねの3倍の重み付け
- **リバーススワイプ（前に戻る）は強い正シグナル**
- **滞在時間（Dwell Time）**: 読むのに時間がかかるスライドは正シグナル
- **まとめスライドの全項目に説明を入れる**: 各項目がちゃんと完結していること
- **手動スワイプ > 自動再生**: 手動スワイプの方が80%高いエンゲージメント
