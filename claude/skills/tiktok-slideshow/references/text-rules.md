## テキスト配置（セーフゾーン内に収める）

ImageMagickの `-gravity center` でY座標オフセットを使う場合、セーフゾーン内に収まるよう注意:

```
セーフゾーン（design-rules.md参照）:
  X: 60〜930px  Y: 130〜1520px

-gravity center 基準（960が中心）でのY座標範囲:
  上限: -830 (= 960 - 130 - 960)  ← これ以上は上にはみ出る
  下限: +560 (= 1520 - 960)       ← これ以上は下にはみ出る

実用的な配置範囲:
  annotate の Y: -550 〜 +450 程度
```

- 1スライドあたりの文字数: 12〜40文字を目安（多すぎると読まれない）
- **全て中央寄せ** (`-gravity center`)
- **stroke none**: 縁取りなし（小さいテキストが潰れる）
- 強調: **ティール色 + 1段階大きいフォントサイズ + W7フォント**
- スワイプ誘導: `スワイプ →`（TikTokは左→右）

## テキストはみ出し防止（最重要）

テキストが左右にはみ出すと投稿として使えない。以下のルールを厳守する:

### 1行あたりの最大文字数（目安）

| pointsize | 全角最大文字数 | 例 |
|-----------|-------------|---|
| 86 | **10文字** | 「体が変わらない人へ」 |
| 76 | **11文字** | 「その人も最初は同じ」 |
| 66 | **13文字** | 「「10回目がギリギリ」」 |
| 60 | **14文字** | 「「ちょっとキツい」が成長のサイン」はNG |
| 54 | **16文字** | 「鏡見ながらゆっくり1レップ」 |
| 50 | **17文字** | 長い説明文向け |
| 44 | **19文字** | 補助テキスト |

### はみ出し防止チェックリスト

- **1行の文字数がpointsizeの上限を超えたら必ず改行する**
- 改行できない場合はpointsizeを下げる（86→76→66→54）
- Q行は文字数が多くなりがち → pointsize 50〜54で安全に
- 「」や（）を含む行は実質+2文字分として計算する
- 半角英数（kg/g/cm等）は全角の約0.5文字分として計算
- **ImageMagickコマンド生成後 必ず文字数チェックを行うこと**
